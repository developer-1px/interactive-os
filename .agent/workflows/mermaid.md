## /mermaid — 다이어그램 전용 턴

> 한 턴에 하나의 관심사. 다이어그램은 다른 작업과 섞지 않는다.

### 절차

1. **의도 추론 → 선행 실행** — 대화 맥락에서 시각화 대상과 목적을 추론한다.
   - 맥락이 충분하면 (Clear/Complicated): 추론한 의도를 한 줄로 밝히고, **바로 그린다**. 완성 후 "이 의도로 그렸습니다. 수정할 부분 있으면 말씀해주세요."
   - 맥락이 불충분하면 (Complex): "무엇의 관계를 보고 싶으신가요?"로 묻는다.
   - ⛔ Clear/Complicated에서 "뭘 그릴까요?"로 시작하는 것은 금지. 맥락을 읽고 먼저 행동한다.

2. **소스 수집** — 다이어그램에 필요한 데이터를 코드/문서에서 수집한다.
   - `view_file_outline`, `grep_search` 등으로 식별자·관계를 추출한다.
   - 이 단계에서 다이어그램을 작성하지 않는다. 데이터만 모은다.

3. **다이어그램 설계** — 수집한 데이터로 Mermaid 코드를 작성한다.
   - 다이어그램 유형 선택: `graph`, `flowchart`, `sequenceDiagram`, `classDiagram`, `stateDiagram`, `erDiagram`
   - 노드 라벨에 특수문자가 있으면 `["label"]`로 감싼다.
   - 하나의 다이어그램이 노드 20개를 넘으면 subgraph로 분할하거나, 여러 다이어그램으로 쪼갠다.

4. **파일 저장** — 다이어그램을 포함한 문서를 파일로 저장한다.
   - 저장 위치: `/inbox` 라우팅 규칙을 따른다.
   - 파일명: `YYYY-MMDD-HHmm-[analysis]-{주제}-diagram.md`
   - 다이어그램만 단독 저장하지 않는다. 제목 + 맥락 설명 + 다이어그램 + 범례를 포함한다.

5. **검증** — 저장된 파일을 사용자에게 알리고, 렌더링 확인을 안내한다.
