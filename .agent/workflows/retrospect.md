## /retrospect — KPT 회고

### 원칙

> 작업이 끝났을 때 **Keep / Problem / Try**로 회고한다.
> 같은 실수를 반복하지 않기 위한 구조화된 기록.

### 절차

1. **컨텍스트 파악**
   - 이번 세션 또는 프로젝트에서 무엇을 했는지 요약한다.
   - 관련 커밋, 변경 파일, 논의를 식별한다.

2. **KPT 작성**

   #### Keep 🟢
   - 잘한 것. 계속 유지해야 할 패턴, 습관, 결정.
   - 구체적인 코드/결정 사례를 포함한다.

   #### Problem 🔴
   - 문제가 된 것. 삽질, 잘못된 가정, 놓친 것.
   - **왜** 문제가 됐는지 원인을 분석한다.

   #### Try 🔵
   - 다음에 시도할 것. Problem의 해결책 또는 개선 아이디어.
   - 실행 가능한 수준으로 구체적으로 작성한다.

3. **산출물**
   - **프로젝트 회고**: `docs/1-project/{name}/6-retrospective.md`에 저장.
   - **세션 회고**: 대화 내에서 인라인으로 보여주거나, 사용자 요청 시 파일로 저장.

### 사용 시점

- `/project` Phase 5 (CLOSING)에서 호출
- 세션 종료 시 사용자가 "회고" 또는 "KPT"를 요청했을 때
- 큰 실수를 한 직후 (즉시 회고)
