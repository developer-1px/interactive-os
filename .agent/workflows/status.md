---
description: PARA 문서(1-project, 2-area) 기반으로 RAG + Kanban 스타일의 프로젝트 진척도 현황 리포트를 생성한다.
---

1. **Scan Projects & Areas**:
   - `docs/1-project/`의 모든 하위 폴더를 스캔하여 프로젝트 목록을 수집한다.
   - `docs/2-area/`의 모든 하위 폴더를 스캔하여 영역 목록을 수집한다.

2. **Analyze Progress Indicators**:
   - 각 문서를 읽고 다음 진척도 신호를 추출한다:
     - 체크리스트: `[ ]` (미완료), `[x]` (완료), `[/]` (진행 중)
     - 상태 이모지: ✅ (완료), ⚠️ (부분/주의), ❌ (미착수)
     - 테이블의 "상태" 컬럼
   - 필요 시 코드베이스와 대조하여 실제 구현 여부를 확인한다.

3. **Generate Status Report**:
   - `docs/0-inbox/YYYY-MMDD-HHmm-[report]-project-status.md`에 리포트를 저장한다.
   - **RAG + Kanban 하이브리드 형식**:
     - **프로젝트 현황 테이블**: 프로젝트명 | RAG 상태(🟢🟡🔴) | Done | In Progress | Todo | 진척률(%) | 비고
     - **블로커 (🔴)**: 즉시 조치가 필요한 항목
     - **주의 항목 (🟡)**: 리스크가 있거나 지연 가능성이 있는 항목
     - **최근 완료 항목**: 최근 변경된 완료 건 목록
     - **영역(Area) 개요**: 각 영역의 문서 수와 최근 갱신일
   - RAG 판정 기준:
     - 🟢 Green: 진척률 70%+ 또는 블로커 없음
     - 🟡 Amber: 진척률 30~70% 또는 부분 완료 항목 존재
     - 🔴 Red: 진척률 30% 미만 또는 블로커 존재

4. **Notify User**:
   - 생성된 리포트를 사용자에게 보여주고 리뷰를 요청한다.
