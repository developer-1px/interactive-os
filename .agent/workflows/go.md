## /go — 자율 주행

> `/go`는 세 가지 뜻이다:
> 1. **"시작해"** — 새 세션. STATUS.md 읽고 뭘 할지 파악.
> 2. **"이어해"** — 하다 만 작업. 마지막 단계에서 재개.
> 3. **"진행해"** — AI가 질문했을 때 "ㅇㅇ 해".
>
> 세 경우 모두: **지금 상태를 파악하고, 다음 할 일을 하라.**

---

## Phase 0: 부팅

1. **Rule 숙지** — `.agent/rules.md`를 읽는다.
2. `docs/STATUS.md`를 읽는다. (Active Focus 프로젝트, 현재 Phase)
3. 해당 프로젝트의 `BOARD.md`를 읽는다. (Now 태스크, 마지막 체크포인트)
4. 대화 맥락이 있으면 참조한다.
5. **"지금 보편 사이클의 몇 번째 단계에 있는가?"** 판단한다.
6. 해당 단계로 점프한다.

새 작업이면 Phase 1(Step 1)부터.

## 보편 사이클 (The One Cycle)

모든 작업은 이 사이클을 따른다.
경중에 따라 단계를 **skip**할 수 있지만, **순서는 바뀌지 않는다.**
`/reflect`는 어떤 단계 뒤에든 삽입할 수 있다.

```
┌─ 시작 ──────────────────────────────────────┐
│  1. /ready        — 환경 보장                │
├─ 생각 ──────────────────────────────────────┤
│  2. /discussion   — 뭘 왜 하는지 생각       │
│  3. /prd          — 요구사항 정의            │
│  4. /redteam      — 공격적 검증              │
├─ 만들기 ────────────────────────────────────┤
│  5. /tdd          — 테스트 먼저              │
│  6. /solve        — 구현 (divide & conquer)  │
├─ 다듬기 ────────────────────────────────────┤
│  7. /review       — 코드 리뷰               │
│  8. /fix          — 형식 정정                │
│  9. /doubt        — 의심                     │
│ 10. /cleanup      — 정리                     │
├─ 확인 ──────────────────────────────────────┤
│ 11. /verify       — 검증 게이트              │
├─ 기록 ──────────────────────────────────────┤
│ 12. /changelog    — 커밋                     │
│ 13. /ready        — 돌아가는지 확인          │
├─ 회고 ──────────────────────────────────────┤
│ 14. /retrospect   — 뭘 배웠나               │
│ 15. /coverage     — 테스트 충분한가          │
│ 16. /para         — 문서 정리                │
└──────────────────────────────────────────────┘
```

## Skip 프리셋

경중에 따라 skip하는 단계가 다르다.
skip은 허용이지, 필수는 아니다 — 필요하면 언제든 실행해도 된다.

| 단계 | Heavy (project) | Medium (issue, refactor) | Light (poc, hotfix) |
|------|:---:|:---:|:---:|
| 1. /ready | ✅ | ✅ | ✅ |
| 2. /discussion | ✅ | skip | 선택 |
| 3. /prd | ✅ | skip | skip |
| 4. /redteam | ✅ | skip | skip |
| 5. /tdd | ✅ | ✅ | skip |
| 6. /solve | ✅ | ✅ | ✅ |
| 7. /review | ✅ | ✅ | skip |
| 8. /fix | ✅ | ✅ | ✅ |
| 9. /doubt | ✅ | 선택 | skip |
| 10. /cleanup | ✅ | 선택 | skip |
| 11. /verify | ✅ | ✅ | ✅ |
| 12. /changelog | ✅ | ✅ | ✅ |
| 13. /ready | ✅ | ✅ | ✅ |
| 14. /retrospect | ✅ | ✅ | skip |
| 15. /coverage | ✅ | 선택 | skip |
| 16. /para | ✅ | skip | skip |

### 프리셋 결정 기준

- `/project` (새 기능, 아키텍처 변경) → **Heavy**
- `/issue` (버그 수정) → **Medium** (2~4 skip, 5부터 시작)
- `/refactor` (패턴 전환) → **Medium** (2~4 skip)
- `/poc` (실험) → **Light**
- 명시적 지정 없으면: BOARD.md 태스크의 Cynefin 도메인으로 판단
  - Complex → Heavy
  - Complicated → Medium
  - Clear → Light

## 진입점 라우팅

사용자 요청의 성격에 따라 보편 사이클의 진입 단계가 다르다:

| 요청 | 진입 단계 | 프리셋 |
|------|----------|--------|
| "이거 해보자 / 만들자" | Step 2 (/discussion) | Heavy |
| "이거 고장났어 / 안 돼" | Step 5 (/tdd — 재현 테스트) | Medium |
| "이거 개선하자 / 리팩토링" | Step 5 (/tdd) | Medium |
| "실험해보자" | Step 6 (/solve) | Light |
| "계속해" | 마지막 체크포인트에서 재개 | 기존 유지 |

## 상태 기록

매 단계 완료 시 BOARD.md에 체크포인트를 기록한다:

```markdown
## 🔴 Now
- [ ] 태스크명 — 한 줄 설명
  - [x] Step 5: /tdd
  - [x] Step 6: /solve
  - [ ] Step 7: /review     ← 다음 재개 지점
```

세션이 끊겨도 다음 `/go`에서 이 체크포인트를 읽고 이어서 진행한다.

## 반복

BOARD.md에 Now 태스크가 여러 개이면, 각 태스크마다 Step 5~13을 반복한다.
모든 Now 태스크가 완료되면 Step 14~16(회고/커버리지/정리)을 1회 실행한다.
