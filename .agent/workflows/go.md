## /go — 자율 주행

> `/go`는 세 가지 뜻이다:
> 1. **"시작해"** — 새 세션. STATUS.md 읽고 뭘 할지 파악.
> 2. **"이어해"** — 하다 만 작업. 마지막 단계에서 재개.
> 3. **"진행해"** — AI가 질문했을 때 "ㅇㅇ 해".
>
> 세 경우 모두: **지금 상태를 파악하고, 다음 할 일을 하라.**

---

## Phase 0: 부팅

1. **Rule 숙지** — `.agent/rules.md`를 읽는다.
2. `docs/STATUS.md`를 읽는다. (Active Focus 프로젝트, 현재 Phase)
3. 해당 프로젝트의 `BOARD.md`를 읽는다. (Now 태스크, 마지막 체크포인트)
4. 대화 맥락이 있으면 참조한다.
5. **"지금 보편 사이클의 몇 번째 단계에 있는가?"** 판단한다.
6. 해당 단계로 점프한다.

새 작업이면 Phase 1(Step 1)부터.

---

## 보편 사이클 (The One Cycle)

모든 작업은 이 사이클을 따른다.
경중에 따라 단계를 **skip**할 수 있지만, **순서는 바뀌지 않는다.**
`/reflect`는 어떤 단계 뒤에든 삽입할 수 있다.

```
┌─ 시작 ──────────────────────────────────────┐
│  1. /ready        — 환경 보장                │
├─ 생각 ──────────────────────────────────────┤
│  2. /discussion   — 뭘 왜 하는지 생각       │
│  3. /prd          — 요구사항 정의            │
│  4. /reflect      — 요구사항 방향 점검       │
│  5. /premortem    — 선제적 실패 분석          │
│  6. /divide       — 분해 보고서              │
│  7. /blueprint    — 실행 설계도              │
│  8. /naming       — 이름 설계                │
├─ 만들기 ────────────────────────────────────┤
│  9. /tdd          — 테스트 먼저              │
│ 10. /solve        — 구현 (divide & conquer)  │
│ 11. /refactor     — 패턴 전환                │
├─ 다듬기 ────────────────────────────────────┤
│ 12. /review       — 코드 리뷰               │
│ 13. /fix          — 형식 정정                │
│ 14. /doubt        — 의심                     │
│ 15. /cleanup      — 정리                     │
├─ 확인 ──────────────────────────────────────┤
│ 16. /verify       — 검증 게이트              │
├─ 기록 ──────────────────────────────────────┤
│ 17. /changelog    — 커밋                     │
│ 18. /ready        — 돌아가는지 확인          │
├─ 회고 ──────────────────────────────────────┤
│ 19. /retrospect   — 뭘 배웠나               │
│ 20. /coverage     — 테스트 충분한가          │
│ 21. /para         — 문서 정리                │
├─ 완료 ──────────────────────────────────────┤
│ 22. /archive      — 프로젝트 완료 시 자동 매장│
└──────────────────────────────────────────────┘
```

## 모든 단계를 통과한다

Skip은 없다. 모든 작업은 모든 단계를 순서대로 통과한다.
LLM에게 "생각을 덜 해도 된다"는 허가는 없다.
단계의 산출물 크기가 다를 뿐이다 — Clear면 1줄, Complex면 문서.

진입점은 항상 Step 2(/discussion)이다. 진입 후 단계를 건너뛰지 않는다.

| 요청 | 진입 |
|------|------|
| "이거 해보자 / 만들자" | Step 2 |
| "이거 고장났어 / 안 돼" | Step 2 |
| "이거 개선하자 / 리팩토링" | Step 2 |
| "실험해보자" | Step 2 |
| "계속해" | 마지막 체크포인트에서 재개 |

---

## Step 16 (/verify) — 완료의 정의

**"완료"는 Red→Green 증명이다.**

1. 수정 **전**: 버그/요구사항을 재현하는 **Red 테스트**를 먼저 쓴다.
   - `createPage(App, Component)`로 실제 렌더링 경로를 포함한 headless 재현.
   - "브라우저에서 확인해주세요"는 금지. 이 OS는 headless 재현이 가능하다.
2. 수정 **후**: 같은 테스트가 **Green**이 된다.
3. 기존 테스트 스위트가 깨지지 않는다 (**regression 없음**).

| 증명 상태 | 보고 유형 | BOARD.md 표기 |
|-----------|----------|--------------|
| Red→Green + regression 없음 | **완료** | `[x] 태스크명 ✅` |
| 수정했지만 Red→Green 없음 | **현황 보고** | `[ ] 태스크명 — 증명 미완` |
| 기존 테스트 깨짐 | **미완료** | `[ ] 태스크명 — regression` |

## 상태 기록

매 단계 완료 시 BOARD.md에 체크포인트를 기록한다:

```markdown
## 🔴 Now
- [ ] 태스크명 — 한 줄 설명
  - [x] Step 9: /tdd
  - [x] Step 10: /solve
  - [ ] Step 11: /refactor     ← 다음 재개 지점
```

세션이 끊겨도 다음 `/go`에서 이 체크포인트를 읽고 이어서 진행한다.

---

## 반복

BOARD.md에 Now 태스크가 여러 개이면, 각 태스크마다 Step 9~18을 반복한다.
모든 Now 태스크가 완료되면 Step 19~22(회고/커버리지/정리/아카이브)를 1회 실행한다.

### Step 22: /archive 자동 실행 조건

프로젝트의 **모든 Now 태스크가 Done**이고, Ideas에 남은 것이 없거나 별도 프로젝트로 분리된 경우:
→ `/archive`를 자동 호출한다.
Ideas가 남아있으면 아카이브 전에 사용자에게 확인한다.
