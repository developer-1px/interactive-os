## Red Team Report

### 1. 주제 파악
- 추가 입력이 없으면 → **직전 작업**(최근 대화 히스토리)을 분석 대상으로 삼는다.
- 추가 입력이 있으면 → 해당 주제에 대해 깊이 분석한다.

### 2. 관련 코드/문서 탐색
- 주제와 관련된 코드, 문서, ADR 등을 탐색하여 맥락을 파악한다.

### 3. 🔴 레드팀 분석 (기본)
- **토론과 논쟁 위주**로 자유롭게 작성한다.
- 설계 결정의 약점, 엣지케이스, 숨은 가정, 대안을 공격적으로 제기한다.
- 코드 예시와 근거를 포함한다.

### 4. 🔵 블루팀 분석 (사용자가 "블루팀"을 언급한 경우에만)
- 레드팀의 공격에 대한 방어 논거를 제시한다.
- 결론 및 합의 가능한 중간안을 도출한다.

### 5. 보고서 저장
- `docs/0-inbox/` 에 `{날짜}_{주제}_RedTeam.md` 형식으로 저장한다.
