## /reflect — 멈추고 돌아보기

> **분류**: 리프. 다른 워크플로우를 호출하지 않는다.
> **특성**: 보편 사이클의 어떤 단계 사이에든 삽입할 수 있는 **cross-cutting 점검**.

### 역할

직전에 수행한 행위를 되돌아본다:

1. **의도 확인** — 방금 한 것이 원래 의도와 맞는가?
2. **부작용 점검** — 의도치 않은 변경이나 손상이 없는가?
3. **방향 확인** — 전체 목표 대비 올바른 방향으로 가고 있는가?

### 절차

```
1. 직전 행위를 1문장으로 요약한다.
2. 3가지 질문에 답한다:
   ① 의도와 결과가 일치하는가?
   ② 건드리지 않아야 할 것을 건드리진 않았는가?
   ③ 다음 단계로 넘어가도 되는가?
3. 문제가 없으면 → "✅ reflect 통과" 한 줄.
   문제가 있으면 → 무엇이 문제인지 보고하고 멈춘다.
```

### 호출 시점

오케스트레이터가 보편 사이클의 **어떤 단계 뒤에든** 삽입할 수 있다.
사용자가 명시적으로 `/reflect`를 호출해도 된다.

```
... → /solve → /reflect → /doubt → ...
... → /tdd → /reflect → /solve → ...
... → /fix → /reflect → /verify → ...
```

### /review, /doubt와의 차이

| | /reflect | /review | /doubt |
|---|---------|---------|--------|
| **무게** | 가볍다 (1분) | 무겁다 (전수 검사) | 중간 (재귀) |
| **질문** | "방금 한 거 맞아?" | "규칙/철학을 위반했나?" | "이거 필요해?" |
| **시점** | 매 단계 후 | 구현 완료 후 | 다듬기 단계 |
| **산출물** | 한 줄 통과/실패 | 위반 보고서 | 제거/축소 보고서 |
