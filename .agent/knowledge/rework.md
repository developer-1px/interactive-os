# REWORKBOOK — Rework 지식베이스

> `/rework` 시작 시 반드시 읽는다.
> "완료라고 했는데 안 됐던" 사례와 근본 원인을 축적한다.
> 같은 유형의 실수를 반복하지 않기 위한 기록.

---

## 1. 실수 유형 분류

| 유형 | 설명 |
|------|------|
| 검증 누락 | 수정 후 실제 동작을 확인하지 않음 |
| 잘못된 가정 | 코드가 이렇게 동작할 것이라고 추측 |
| 부분 적용 | 여러 파일 중 일부만 수정 |
| 사이드이펙트 | 수정이 다른 곳을 깨뜨림 |

---

## 2. Rework 선례

|------|----------|----------|-----------|
| (아직 축적된 사례 없음) | | | |

## 갱신 방법

`/rework` 종료 시:
1. Step 6의 회고 한 줄을 §2 테이블에 추가한다.
2. 새로운 실수 유형이 발견되면 §1에 추가한다.
