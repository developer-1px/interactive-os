# BINDBOOK — UI 바인딩 지식베이스

> `/bind` 시작 시 반드시 읽는다.
> Green 로직을 UI에 연결할 때 반복되는 패턴과 실수를 축적한다.
> 새 지식이 생기면 `/bind` 종료 시 이 파일을 갱신한다.

---

## 1. 바인딩 원칙

- **OS Hook 우선**: raw HTML 이벤트(`onClick`, `onKeyDown`) 대신 OS가 제공하는 Hook/Command를 사용한다.
- **RUNBOOK 참조**: OS 패턴의 상세는 `.agent/knowledge/runbook.md`를 본다.
- **Passive Projection**: 앱은 상태를 투영만 하고, 행동은 OS가 실행한다.
