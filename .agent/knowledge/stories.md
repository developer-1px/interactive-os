# STORIESBOOK — User Story 지식베이스

> `/stories` 시작 시 읽는다. 종료 시 새 지식이 있으면 갱신한다.
> 좋은 Story의 패턴과 함정을 축적하여 반복 실수를 방지한다.

---

## 1. 알려진 좋은 패턴

| 패턴 | 설명 | 발견일 |
|------|------|--------|
| **INVEST S = 파이프라인 1사이클** | Story 크기는 spec→red→green→bind→audit 한 바퀴가 돌아야 함. 더 크면 분해하라 | 2026-02-25 |
| **UX Flow 필수** | Story는 반드시 "화면 어디서 무엇이 보이고, 어떤 입력이 있고, 화면이 어떻게 바뀌는가"를 포함해야 함 | 2026-02-25 |
| **DT Gate** | DT(Decision Table)를 만들 수 없는 Story는 개발 불가. 기획으로 돌려보내라 | 2026-02-25 |
| **수직 분해** | "언어 전환" = 하나의 수직 슬라이스. "데이터 모델 변경"은 Story가 아님 | 2026-02-25 |
| **DT는 Story에 포함** | DT는 Product 자산 → `6-products/`에 Story와 함께 저장. `/spec`에서 새로 쓰지 않는다 | 2026-02-25 |
| **DnD = OS onReorder** | Drag & Drop은 OS가 제공. `Zone.onReorder` 콜백으로 선언. raw pointer 이벤트 사용 금지 | 2026-02-25 |

## 2. 알려진 함정

| 함정 | 결과 | 대응 |
|------|------|------|
| **수평 분해** | T1=데이터모델, T2=UI 식으로 나누면 DT가 없고 파이프라인 검증 불가 | 수직 분해로 재작성 |
| **추상 AC** | "액션을 실행한다" → DT 불가. "🌐 KO ▾ 버튼을 클릭한다"로 구체화해야 함 | UI 요소+물리적 입력으로 재작성 |
| **너무 큰 Story** | US-001에 언어전환+추가+독립편집 모두 포함 → 3 사이클 분량 | 1사이클 단위로 분해 |

## 갱신 방법
`/stories` 세션 종료 후:
- 새 INVEST 기준 발견 → §1 추가
- 반복된 함정 → §2 추가
- 거부/승인 결정 내린 패턴 → §3 추가
