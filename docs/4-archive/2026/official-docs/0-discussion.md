# Discussion: 공식 문서 분리

> 2026-02-14 세션에서 합의된 내용

---

## 배경

커널이 프리징된 상태에서, 외부 공개용 공식 문서가 필요한지 점검했다.

## 발견

1. `docs/2-area/05-kernel/`에 이미 **10개 문서, ~61KB**의 체계적인 기술 문서가 존재
2. 코드-문서 정합성 검증 결과: **모든 API가 1:1 일치**
3. 문서 품질: **8.5/10** — 거의 공식 문서 수준
4. 문제: PARA 시스템 안에 묻혀 있어 외부 사람이 찾기 어려움

## 합의된 방향

- **PARA 내부에 `docs/official/` 폴더를 만들어 분리**
- 번호 접두사 없이 PARA 체계 밖에 위치
- 나중에 `website/`나 `site/`로 통째로 이동 가능하도록 자립형 구조
- `docs/2-area/05-kernel/`의 공식 급 문서 10개 → `docs/official/kernel/`로 이동
- `kernel-adr-journey.md`는 내부 히스토리이므로 `2-area/05-kernel/`에 잔류
- 문서 내 `file:///` 링크 → 상대 경로로 수정

## 논의된 대안

| Option | 설명 | 판단 |
|--------|------|------|
| A. `docs/` 분리 | 공식 문서가 `docs/`를 차지, PARA는 `.docs/`로 | ❌ 대량 이동 필요 |
| B. 별도 최상위 폴더 | `website/` 등 새 폴더 | ❌ 아직 이르다 |
| B' (선택) | `docs/official/` — PARA 내부 분리, 나중에 이동 | ✅ 채택 |
| C. 패키지별 문서 | `packages/kernel/docs/` | ❌ 구조 복잡 |
