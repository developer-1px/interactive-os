## 분류 근거

### #1 arrowExpand 잔류 (🔴)
- T2에서 chain executor가 arrowExpand를 완전 대체함
- 하지만 `NavigateConfig.arrowExpand: boolean` 필드, `DEFAULT_NAVIGATE.arrowExpand: false`, tree/treegrid preset의 arrowExpand: true, navigate/index.ts의 legacy fallback 코드가 모두 남아있음
- **근본 원인**: Green 구현 시 backward compatibility를 위해 남겼으나, 제거 결정 후 실행 안 함
- **진단**: Bind 잘못됨 → `/refactor` (dead code 제거)

### #2 OS_ACTIVATE expandableItems (🟡)
- `activate.effect: "toggleExpand"`가 rolePreset에 선언되었지만, OS_ACTIVATE 내부에서는 이 config를 읽지 않음
- 대신 `ctx.inject(DOM_EXPANDABLE_ITEMS)`로 DOM 기반 검사 수행
- 결과적으로 동작은 하지만, config-driven이 아닌 DOM-driven → "command-config invariant" 계약 위반
- **행선지**: BOARD Unresolved → config-driven activate.effect 소비 구현 필요

### #3 dead test (🔴)
- resolve-item-key.test.ts에서 treeitem resolver가 OS_EXPAND를 반환하는 테스트가 남아있음
- T3에서 resolver를 제거했으므로 이 테스트는 dead → 실제로는 다른 경로로 통과하거나 실패 상태
- **진단**: Red 테스트 잘못됨 → 삭제 또는 chain 경로로 재작성
