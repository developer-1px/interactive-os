| 항목 | 내용 |
|------|------|
| 원문 | 보편적인 이슈 관리나 프로젝트 관리 문서 관리등의 툴에서 사용되는 표준적이거나 보편적인 기능들을 한번 조사해올래? |

## 1. 개요 (Overview)

Notion, Obsidian, Confluence, Linear, Jira, GitHub Issues 등 주요 문서/프로젝트/이슈 관리 도구들의 표준 기능을 조사하고, 우리 docs-viewer에 대한 적용 가능성을 분석한다.

현재 docs-viewer는 **폴더 기반 트리 네비게이션 + 마크다운 렌더링 + 외부 폴더 열기 + Recent 섹션(mtime 기반)**을 갖추고 있다.

## 3. 우선순위 제안 (Proposal)

**투자 대비 효과(ROI)가 높은 순서**로 분류:

### 🟢 Tier 1 — 저비용 고효과 (즉시 가능)

| 기능 | 근거 | 예상 공수 |
|------|------|----------|
| **A9. TOC (목차)** | 마크다운 heading 파싱으로 구현 가능. 긴 문서 탐색에 필수. | 2-3시간 |
| **A3. Favorites / Pinned** | localStorage + sidebar 섹션 추가. Recent과 동일 패턴. | 2시간 |
| **B4. 날짜 메타데이터 UI** | mtime 인프라 이미 있음 (virtual:docs-meta). 문서 헤더에 표시만 추가. | 30분 |

### 🟡 Tier 2 — 중간 비용 높은 효과

| 기능 | 근거 | 예상 공수 |
|------|------|----------|
| **D1/A4. 전문 검색** | 모든 문서가 이미 `import.meta.glob(?raw)`로 로드 가능. 클라이언트 사이드 검색 (예: Fuse.js) 충분. | 4-6시간 |
| **D4. Command Palette** | Cmd+K로 문서 quick-switch. 검색과 결합 가능. | 3-4시간 |
| **C7. 내부 문서 링크** | `[[doc-name]]` 패턴을 마크다운 렌더러에서 해석. 문서 간 연결의 시작. | 3시간 |

### 🔴 Tier 3 — 고비용 또는 맥락 의존

| 기능 | 근거 | 비고 |
|------|------|------|
| **A6. 백링크** | 내부 링크 (C7) 선행 필요. 전문 검색 인프라 위에 구축. | C7, D1 의존 |
| **B1. 태그** | YAML frontmatter 파싱 필요. 현재 문서에 frontmatter 관행 없음. | 워크플로우 변경 필요 |
| **C4. 인라인 편집** | 읽기 전용 뷰어의 정체성과 충돌. 편집은 IDE에서. | 의사결정 필요 |
| **A7. 그래프 뷰** | 내부 링크 + 백링크 기반. 시각적 효과는 크나 실용성 대비 공수 큼. | C7, A6 의존 |

## 4. Cynefin 도메인 판정

🟢 **Clear** — 이 기능들은 모두 30년간 확립된 문서 관리 표준이다. 정답이 명확하고, 각 도구가 이미 최적해를 증명했다. 구현 방법도 잘 알려져 있다. 남은 것은 "어떤 순서로, 어느 것을 하느냐"의 우선순위 결정뿐이다.

## 5. 인식 한계 (Epistemic Status)

- 이 분석은 **공개된 제품 문서와 비교 리뷰** 기반이며, 각 도구의 최신 업데이트(2026년 초)는 일부 반영되지 않았을 수 있다.
- 공수 추정은 현재 docs-viewer 코드베이스 파악 기반이나, **실제 구현 시 예상치 못한 제약이 있을 수 있다**.
- 1인 프로젝트 맥락에서 **협업 기능(B3, C6 등)은 ROI가 낮다**고 판단했으나, 향후 팀 확장 시 재평가 필요.

## 6. 열린 질문 (Complex Questions)

1. **검색(D1)과 Command Palette(D4)를 별도로 만들 것인가, 하나로 합칠 것인가?** — Obsidian은 합쳤고(Quick Switcher), Notion은 분리(Search vs Quick Find). docs-viewer 규모에서는 합치는 것이 엔트로피를 줄일 수 있다.

2. **TOC(A9)의 위치** — 우측 사이드바 (Confluence 스타일) vs 좌측 사이드바 내 접이식 (Obsidian 스타일) vs 문서 내 상단 고정. 화면 공간 예산에 따라 다르다.

3. **이 기능들을 docs-viewer 전용으로 만들 것인가, OS 앱으로 만들 것인가?** — docs-viewer는 현재 독립 Vite 앱(port 4444)이지만, 이 기능들이 충분히 성숙하면 OS 위에서 돌아가는 앱으로 전환할 수도 있다.

---

> **한줄요약**: Docs viewer에 빠진 표준 기능 중 TOC, Favorites, 전문 검색, Command Palette가 ROI가 가장 높고, 기존 mtime 인프라와 마크다운 로더 위에 저비용으로 구현 가능하다.
