# Discussion — 폴더 클릭 시 하위 페이지 목록 표시

## Journey

**🧑 사용자**: docs viewer에서 폴더 클릭시 하위 페이지 목록이 보이게 하고 싶어

**🤖 AI**: Intent 추론 — 현재 폴더 클릭은 expand/collapse만 하고 메인 영역은 무변화. 폴더가 "도착점"이 되지 못하는 dead-end. 형태 질문: 카드 그리드? 자동 네비게이션? README 인덱스?

---

**🧑 사용자**: 일단은 목록화? 3개 이하면 다 보여주고?

**🤖 AI**: Progressive Disclosure 패턴 확인. "3개 이하"의 기준이 목록인지 내용인지 확인 필요.

---

**🧑 사용자**: 목록은 다 보여주는거야. 내용도 보여주냐 마냐가 3개기준

**🤖 AI**: 이해 — 목록(제목)은 항상 전부, 마크다운 본문 인라인은 ≤3개일 때만. Aggregated Index Page 패턴. 하위 폴더 재귀 여부 질문.

---

**🧑 사용자**: 아니 폴더 포함 1depth로만

**🤖 AI**: 직계 자식만, 폴더+파일 모두 포함. 드릴다운 네비게이션 구조 확정.

---

**🧑 사용자**: 일단 해보자 3개 정도면 그리고 목차가 있으니 클릭해서 보면 그만일테니까

**🤖 AI**: ≤3개면 전체 내용 인라인 표시. TOC가 있으니 긴 문서도 네비게이션 가능.

## Conclusion

| Toulmin | 내용 |
|---------|------|
| **🎯 Claim** | 폴더 클릭 시 메인 영역에 **폴더 인덱스 페이지**를 표시한다. 직계 자식(1-depth, 폴더+파일 모두) 목록을 항상 보여주고, 직계 자식이 ≤3개이면 각 파일의 마크다운 본문도 인라인 표시한다. |

| **🔗 Warrant** | 폴더가 네비게이션의 도착점이 되어야 사용자가 문서 구조를 탐색할 수 있다 |
| **📚 Backing** | GitBook Category Page, Docusaurus Category Index, Stripe Docs 섹션 랜딩 — 모두 폴더에 인덱스 페이지 제공 |
