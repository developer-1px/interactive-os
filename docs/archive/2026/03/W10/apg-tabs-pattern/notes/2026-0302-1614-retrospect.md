## 🔧 개발 과정 (Development)

**Keep 🟢**
- ARIA role 전수 대조 (15개 role)로 "activate = select" 시맨틱 검증. 한 패턴에 맞추고 끝내지 않았음
- `contentRoleMap`/`contentVisibilityMap` — 기존 `childRoleMap`과 동형 패턴으로 자연스러운 확장
- headless 테스트 선행으로 3개 새 테스트 추가 후 브라우저 확인

**Problem 🔴**
- 3연속 오진단: `selectedItems` 추측, `useZoneContext()` 오용, `activate.mode` mismatch
- 원인: 코드의 실제 모양을 보기 전에 머릿속 모델로 코딩

**Try 🔵**
- OS 내부 state 접근 시 반드시 `view_file`로 실제 타입/필드 확인 후 코딩
- 이 교훈은 rules.md "정확성" 규칙에 이미 있으므로 추가 불필요

**🪞 자가 점검**: 3번 오진은 실제로 시간을 많이 낭비. LLM 교정보다 "OS가 잘못된 접근을 구조적으로 차단"하는 방향이 맞음 (discussion에서 확인)
