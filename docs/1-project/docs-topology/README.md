# docs-topology — 문서 시스템 재구성

## WHY

AI 협업에서 10일간 383개 문서가 생산되었지만, 시간이 지나면 맥락을 잃은 파편이 된다.
PARA는 "작업의 지도"이지 "지식의 지도"가 아니어서, 문서 노화를 막는 구조가 부재하다.

## Goals

1. **살아있는 문서**(`official/`) = 소스코드 토폴로지 기반 공식 지식 지도
2. **죽은 문서**(`archive/연도/월/주차`) = 분류 없이 시간순 매장
3. **인큐베이터**(`2-area/`) = official로 아직 졸업 못 한 문서만 잔류
4. 사후 환류 파이프라인: 작업 완료 → official 추출 + archive 매장

## Scope

### In
- archive 구조를 연도/월/주차로 변환
- OS 문서(SPEC.md 등)를 official/os/로 승격
- 2-area/ 정리 (official로 졸업할 것 승격, 나머지 축소)
- /archive 워크플로우에 환류 파이프라인 반영

### Out
- official/kernel/ (이미 완성)
- 코드 변경 (문서 구조만)
- 새 문서 작성 (기존 문서 재배치)
