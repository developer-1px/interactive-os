# Rules 간 우선순위 체계

## 배경

sense-purity 프로젝트에서 발견: AI가 Rule #8(리팩토링 지표 단조 개선)을 근거로 Rule #7(편의보다 명시적)과 Rule #9(이름은 법이다)을 면제하려 함. 사용자가 "이름이 법인가?"라는 질문으로 교정.

## 문제

rules.md의 원칙들이 충돌할 때 어느 것이 우선하는지 명시되어 있지 않다. 에이전트가 편의적인 원칙을 골라 근본 원칙을 무시할 수 있다.

## 잠정 판단

- **구조적 원칙** (#7 명시적, #9 이름, #1 엔트로피) > **전술적 원칙** (#8 리팩토링 비용)
- 이유: 구조를 지키는 비용은 일시적이지만, 구조를 무시한 부채는 영구적
