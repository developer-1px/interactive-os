## Conclusion

### Why
에이전트 산출물(docs, discussions, journals 등)의 **소비 경험**을 개선하고 싶다. 현재 시스템에 만족하지만, 더 나은 방식으로 읽고, 찾고, 변화를 감지하고 싶다.

### Intent
기존 docs 시스템을 **세 가지 축**으로 확장한다:
1. **검색** — CommandPalette + fuzzyMatch로 docs 파일 검색 (`Shift+Shift`, `Cmd+K`)
2. **페이지네이션** — 기존 문서 뷰에서 ←/→ 키보드로 페이지 단위 읽기
3. **알림** — Vite HMR 기반 파일 변경 감지, GlobalNav에 red dot

### Warrants
- W1. 새로 만드는 것이 아니라 **기존에 얹는 것** → 과잉 엔지니어링 방지
- W2. **Dogfooding** — 만드는 사람 = 쓰는 사람 → 프레임워크 검증 + 도구 확보
- W3. 검색은 새 앱이 아니라 CommandPalette의 자연스러운 확장
- W4. 페이지네이션은 별도 앱이 아니라 기존 뷰의 **읽기 모드** 추가
- W5. 알림은 Vite HMR 인프라에 올라탐, 별도 watcher 불필요
- W6. 분할 기준: `---`/`#` hard break, 테이블/코드블록 atomic, height estimation + 약간의 오차 허용

### 한 줄 요약
**기존 docs 시스템에 검색·페이지네이션·알림을 얹어, 에이전트 산출물의 소비 경험을 자연스럽게 향상시킨다.**
