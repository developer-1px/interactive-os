## /premortem — 선제적 실패 분석

> Premortem (Gary Klein, *The Power of Intuition*).
> "프로젝트가 처참하게 실패했다. 왜인지 써라."
> 블루팀 방어 없이, 공격만 한다. 수정 의무 없음 — 기록용.

### 시점

`/go` 보편 사이클 Step 4. PRD 작성 직후, 구현 전에 실행.

### 절차

1. **대상 확인** — 해당 프로젝트의 PRD, README, BOARD.md를 읽는다.
2. **실패 시나리오 도출** — "이 프로젝트는 이미 실패했다"는 전제에서 출발.
   카테고리별로 나눈다:
   - 🏗️ **구조적** — 아키텍처, 의존성, 확장성
   - 🧠 **인지적** — 복잡도, 학습 곡선, 유지보수 부담
   - 🔄 **진화적** — 요구사항 변경, 기술 부채 누적
3. **위험도 판정** — 각 시나리오에 발생 가능성(H/M/L) × 영향도(H/M/L) 부여.
4. **기록** — `docs/1-project/[name]/premortem.md`에 저장.
   보편 사이클의 다음 단계로 넘어간다 (블루팀 방어 없음).

### `/redteam`과의 차이

| | `/premortem` | `/redteam` |
|---|---|---|
| **시점** | 구현 전 (보편 사이클 Step 4) | 아무 때나 (discussion 중, 독립) |
| **프레임** | "이미 실패했다. 왜?" | "이 결정의 약점은?" |
| **블루팀** | ❌ 없음 | ✅ 있음 |
| **저장** | `1-project/[name]/premortem.md` | `0-inbox/` 또는 인라인 |
| **목적** | 위험 등록부 | 토론/검증 |
