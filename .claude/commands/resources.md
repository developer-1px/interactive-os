## 1. 컨텍스트 파악

현재 무엇을 작업 중인지 자동으로 분석한다.

- `docs/1-project/` 스캔 → 진행 중인 프로젝트 확인
- `docs/2-area/` 스캔 → 관심 영역 확인
- `docs/0-inbox/` 최근 파일 확인 → 최근 이슈/방향 파악
- 최근 git log/diff 확인 → 실제 작업 중인 코드 영역 파악

## 2. 중복 체크

- `docs/3-resource/` 기존 문서를 스캔하여 이미 다루고 있는 주제는 제외한다.

## 3. 1개 주제 선정

- 위 분석을 바탕으로 지금 가장 도움이 될 주제 **1개만** 선택한다.
- 긴급도와 관련도를 기준으로 판단한다.

## 4. 리소스 수집

선정한 주제에 대해 웹 검색으로 깊이 있는 자료를 수집한다:

- 공식 문서 / 스펙
- Best Practice + Anti-Pattern
- 오픈소스 코드 예시
- 설계 철학, 역사, 흥미로운 이야기들

## 5. 문서 작성

아래 구조로 스토리가 있는 기술 에세이를 작성한다. frontmatter에 `last-reviewed` 날짜를 포함한다.

```markdown
---
last-reviewed: YYYY-MM-DD
---

# [주제 제목]

> 한 줄 요약

## 왜 이 주제인가
프로젝트 현황과 연결하여 이 주제를 고른 이유를 설명한다.

## Background / Context
이 기술/개념이 등장한 배경, 해결하려는 문제.

## Core Concept
핵심 개념을 깊이있게 설명한다.

## Usage
실제 사용법, 코드 예시를 포함한다.

## Best Practice + Anti-Pattern
해야 할 것과 하지 말아야 할 것을 정리한다.

## 흥미로운 이야기들
설계 철학, 역사, 논쟁, 관련 인사이트 등.

## 📚 스터디 추천
더 깊이 파고 싶을 때 볼 자료를 제안한다.
| 주제 | 이유 | 자료 | 난이도 | 시간 |
```

## 6. 저장

- `docs/3-resource/`에 적절한 파일명으로 저장한다.
- 기존 하위 폴더가 맞는 곳이 있으면 해당 폴더에 저장한다.

## 7. 역링크 연결

- 생성한 리소스와 관련된 `docs/1-project/` 또는 `docs/2-area/` 문서에 리소스 참조 링크를 추가한다.

## 8. 결과 보고

- 생성한 리소스를 사용자에게 알린다.
- "더 필요하면 `/resources` 다시 호출" 안내를 포함한다.
