## /redteam — 외부 관점 자문

### 원칙

> 최대한 공격적으로 반론을 제기하되, **수정 의무는 없다** — 참고용.
> `/discussion` 중 호출 가능. 독립 실행도 가능.

### 프레임

"이 결정의 약점은?" — 설계 결정, PRD, 코드 리뷰에 대한 공격적 반론.

### 1. 주제 파악
- 추가 입력이 없으면 → **직전 작업**을 분석 대상으로 삼는다.
- 추가 입력이 있으면 → 해당 주제에 대해 깊이 분석한다.

### 2. 관련 코드/문서 탐색
- 주제와 관련된 코드, 문서, ADR 등을 탐색하여 맥락을 파악한다.

### 3. 🔴 레드팀 분석
- **토론과 논쟁 위주**로 자유롭게 작성한다.
- 설계 결정의 약점, 엣지케이스, 숨은 가정, 대안을 공격적으로 제기한다.

### 4. 🔵 블루팀 분석
- 레드팀 분석 완료 후 **자동으로 "블루팀 방어를 진행할까요?"라고 묻는다.**
- 사용자가 승인하면 방어 논거를 제시하고, 결론 및 합의표를 도출한다.

### 5. 반환
- discussion 내 호출 시 → 분석 결과를 인라인으로 반환하고 discussion을 이어간다.
- 독립 실행 시 → `docs/0-inbox/YYYY-MMDD-HHmm-[analysis]-{topic}-redteam.md`에 저장한다.
